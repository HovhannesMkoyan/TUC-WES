1. Introduction

Hovhannes Mkoyan

JavaScript is one of the most popular programming languages for creating advanced, interactive and easily maintainable websites. During the last decade, many front-end libraries and frameworks have been built on top of JavaScript, making creating, designing and maintaining interactive websites easier. Those frameworks are the well-known React, Vue.js and the relatively new Svelte.

In this report, these three technologies are discussed one by one, are compared and finally, the comparison results are discussed. The criteria for evaluation are mainly based on the speed of completing certain operations related to DOM elements in the web browser, such as DOM element addition, editing and removal. Evaluation criteria that are non-technical include things like variations in architecture, development experience, popularity, maturity and availability.

1.1. Framework vs Library

Hovhannes Mkoyan

The technologies that will be discussed can be defined with one or two of the following terms: framework and library.
The terms library and framework refer to different things; however, they are very often interchanged with each other in the same context. Before getting into the key differences between these words, there is a common purpose that they both serve. Libraries and frameworks are some codes written by developers to solve complicated problems efficiently. They both give us a better approach to writing DRY (don't repeat yourself) code [7].
However, a framework tells what to do. It is opinionated, which means it has the right way of doing things and provides tooling to support developers. Some of the tools can be:

  1. State management
  2. Handling forms
  3. Routing
  4. HTTP
  5. Testing

Two examples of a framework are Vue.js and Angular.
On the other hand, libraries are utilities built for a singular purpose. For example,

  1. React creates UIs
  2. Redux provides state management
  3. JQuery provides cross-browser DOM manipulation

The technical difference between a framework and a library lies in a term called inversion of control (IoC). As can be seen in Figure 1, when using a library, a user is in charge of the flow of the application. The user chooses when and where to call the library. When using a framework, the framework is in charge of the flow. It provides some places for a user to plug in his/her code, but it calls the code plugged in as needed [7].

2. React

Hovhannes Mkoyan

React (also known as React.js) is a JavaScript library developed by Facebook. React was first released in May 2013. React has a narrow scope and single focus, which is rendering the application user interface in an efficient way using declarative code. As a result, it has a lightweight structure and is less costly to learn and use. Quite often, React is called a framework, although it is a library, and the reason for that is that it is a direct competitor of top JavaScript frameworks like Vue.js and Angular.

React has a component-based architecture. A component is one of the main building blocks of React. Every application one will develop in React will be made up of pieces called components which are separate blocks that act independently of each other. In this way, components make the task of building UIs much easier.

Every React component is made of two main ingredients: state and props.
A state is a built-in object that contains all the data or information about a component. A component's state can change over time, and whenever it does, the component re-renders as a result. State change can happen as a response to user action or system-generated events, and these changes determine the component's behavior and how it will render.
As can be seen in Figure 2, props are used to pass data from one component to another (from a parent component to a child component).

In addition, every component has "lifecycle methods." They specify the behavior of a component when it undergoes different phases of its lifecycle. Some examples of different phases could be: when the component is just about to render on the screen, when it has rendered on the screen, or when it is updated/modified in response to a request or a user-triggered action.

2.1. Virtual DOM

Hovhannes Mkoyan

Virtual DOM (VDOM) is a programming concept where a "virtual" representation of a UI is kept in memory and synchronized with the "real" DOM. This process is called reconciliation. This approach enables the declarative API of React, which means one can tell React what state the UI to be in, and it makes sure the DOM matches that state. This abstracts out the attribute manipulation, event handling and manual DOM updating that would be otherwise used to build an application [8].

The two React ingredients, state and props, let to organize components in a hierarchical structure or a tree that ensures a one-way data flow.
As mentioned, React never updates the original DOM directly (unless a developer use case requires it). Figure 3 shows the reconciliation process through which React updates the DOM.

  1. First of all, React creates a copy of the original DOM, the Virtual DOM. Each of the nodes (circles) of the Virtual DOM represents an element or component.
  2. Next, a new Virtual DOM is created if there is a state update of an element.
  3. The diffing algorithm identifies what has changed. In this case, new nodes “d” and “e” are being added to “c,” so the “c” node has been identified as the difference or the diff.
  4. Lastly, React runs a batch update to update the Original DOM with these changes to keep it in sync.

3. Svelte

Hovhannes Mkoyan

Like React, Svelte is a component-based library too, however it also acts as a compiler and it was created by Rich Harris and released in November 2016. It provides a different approach to building web apps than some of the other frameworks covered. While like React and Vue.js do the most significant part of their work in a browser while the application is running, Svelte shifts that work into a compile step that happens only when a user builds the application, producing highly optimized vanilla JavaScript. As a result, the application needs no dependencies to start and no overhead framework code is injected into the browser. That is why Svelte bundle size leaves a far smaller footprint than other frameworks.

This approach not only produces smaller application bundles and better performance but also has a better developer experience. Like React, Svelte too has a component-based architecture. A Svelte component may contain three sections: script, style and template. The first two, which modify the component's appearance and behavior, are optional, but the template section is required. So, Svelte sticks to the classic web development model of HTML, CSS and JS, just adding a few extensions to HTML and JavaScript. As a result, it arguably has fewer concepts and tools to learn than some of the other framework options.
Svelte extends vanilla web technologies in the following ways:

  1. It extends HTML by allowing the use of JavaScript expressions in markup and providing directives to use conditions and loops in a fashion similar to handlebars.
  2. It extends CSS by adding a scoping mechanism, allowing each component to define its own styles without the risk of clashing with other components' styles.
  3. It extends JavaScript by reinterpreting specific language directives to achieve true reactivity and ease component state management [9].

Svelte compiles the code, so it does not have to use a virtual DOM, bringing some performance improvements. It surgically updates the DOM without relying on an intermediary or complex reconciliation technique. Svelte's compiler tracks changes in variables and updates the HTML accordingly. It looks through a code, watches for components that depend on variables and then updates those components as the variables change. This way, Svelte is reactive without relying on a third-party API.

4. Vue.js

Zohaib Arshad

Vue.js is a growing progressive front-end framework for JavaScript, and it is inspired by Angular Js and it is mainly used for building front-end user Interfaces. Vue.js was initially released by Evan You in 2014. Vue.js can be used to develop complex front-end designs and user interfaces when it is combined with modern tools and libraries. It is referred to as an MVVM (Model View - View Model)-patterned framework since it is built on the idea of two-way data binding to components and views [6].

Vue was created in combination with other well-known technologies like type script. Its meticulously designed to keep components from over-rendering, resulting in an application with excellent performance [6].

In short, Vue is a user-friendly, effective, and flexible framework builds on top of conventional HTML, CSS, and JavaScript and offers an easy-to-use API and excellent documentation. Vue.js has a very good ecosystem which start from a library and can be a fully functional framework. It is a compiler-optimized rendering system that rarely needs manual improvement. You can start with the core library and add on the pieces, you need from its interconnected eco system for example

  1. Static site generation
  2. Vue Router
  3. State Management
  4. IDE Support
  5. Testing Utilities
  6. Build Toolchain
  7. Server-Side Rendering
[6]

4.1. Component-based architecture

Zohaib Arshad

An architecture based on components is used by Vue.js. Components in Vue are essentially like widgets. By using them, we can create reusable custom elements with the appropriate functionality. They can be divided into separate files or can be written in a single file with .vue extension. They are only objects that may have any or all the parameters that the root or any Vue instance may possess, including the ability to render an HTML template [6].

Vue components consist of:

  1. HTML markup: the component's template
  2. CSS style: How the HTML elements are displayed in the webpage
  3. JavaScript code: interactivity and data

4.2. Component Types

Zohaib Arshad

In Vue.js components can be registered as global components, local components

  1. Global Components: Anywhere in the Vue application that supports Global Components
  2. Local Components: Local components can be used inside the parent component where they are created
  3. Fragment component: If your component is a fragment, you can receive a console error alerting you that you cannot do something. Simply include your component template inside a single tag, such as div, to resolve this kind of problem.

4.3. Vue.js Features

Zohaib Arshad

4.3.1. Virtual DOM

Vue.js uses the virtual DOM which is a lite version of HTML DOM with enhanced capabilities, Vue.js uses a clone of the primary DOM. As a result of its superior intelligence than the standard DOM, the virtual DOM finds a way to be more productive. To avoid rendering the entire DOM again after any new changes or updates to the document, it primarily achieves this goal by using a variety of different methods. Because of how infrequently the DOM API is used, this alone greatly improves efficiency and resource management. Changes made through the virtual DOM are much faster than the Real DOM [6].

4.3.2. Data Binding

Two-way data binding, which Vue acquired from Angular, provides a benefit in DOM operations as well. A link between model data updates and views is called two-way data binding (UI). Data that can be occasionally updated is present in bound components. Update connected components and keep track of data changes are made simpler with the aid of two-way data binding [6].

A great feature of Vue for any application demanding real-time updates is the reactive updating of both bound data and DOM objects. The responsiveness of Vue will simplify and speed up data updating for developers. With the help of data binding, HTML attributes can be updated, and values can be assigned to them. Vue.js directive v-bind can also assign classes and change the style of elements.

4.3.3. CSS Transitions and Animations

With the help of Vue.Js' automated transition mechanism, you may add or delete items from the DOM and automatically apply transition effects. Vue.Js will automatically add/remove CSS classes at the proper intervals to trigger CSS transitions or animations, and you can even specify JavaScript hook methods to carry out specific DOM changes throughout the transition.

The same principles that apply to CSS transitions also apply to CSS animations, with the exception that v-enter is deleted after an animation finish event rather than immediately after the element is placed. The transition effect is created by wrapping the element with the built-in transition component provided by Vue.Js. The transition may be applied in several different ways using Vue.Js when HTML elements are added, altered, or deleted from the DOM [6].

4.3.4. Templating

A DOM-based templating implementation is used by Vue.Js. In essence, all Vue.Js templates are legal, parsable HTML that has been given a few unique characteristics. Remember this since it significantly distinguishes Vue templates from string-based templates [6].

4.3.5. Complexity

Vue.js is easier if it is being compared with React.s in terms of design and API. It usually takes less than a day to learn enough to create non-trivial apps, but this is not the case with React.js. React.js has a longer learing path to understand the basic concept and implement it in the real application [6].

4.4. Pros and Cons of Vue.js

Zohaib Arshad

4.4.1. Pros

The most important factor of Vue.Js' superiority over other frameworks is its size. It is significantly simpler to download and utilize because its size is in the 18–21 KB range. It is one of the fastest client-side libraries because to its Virtual DOM implementation.

Vue.js is quick and simple to learn because of its comprehensive and in-depth documentation. It enables the creation of apps by inexperienced developers who only have a working understanding of HTML and JavaScript. This demonstrates that a novice can handle complex procedures with little to no resources and have them succeed.

Vue.js's responsiveness is one of its finest qualities. It has never been simpler to link data between HTML and JS code. Two-way reactive data-binding is handled by Vue quite effectively. The DOM is updated in line with changes to data. Any application that needs real-time updates and SPAs would benefit greatly from it [6].

4.4.2. Cons

Vue.Js is a fairly new framework. Its community and development teams are still too small to be compared with those of more established frameworks. Large businesses do not financially support it either. A stable and well-supported technology will be more likely to be used in large-scale projects since problems will be resolved more rapidly. Even though Vue does not have many issues with it and is even in demand from businesses like IBM and Adobe, it is still utilized in mid-sized projects.

The flexibility of Vue.Js in large projects with several developers becomes a problem that may be solved in the presence of a large community but Vue.Js could occasionally make problems worse and make the project more difficult. It is convenient to have a lot of alternatives and flexibility within the confines of the framework you are using. This might be a problem, though, if more people are working on a larger project. Overly flexible design can complicate a project, increase the number of bugs and inconsistencies in the code, cause a delay in the project, and raise the cost of development.

4.5. Code Complexity and learning curve

Zohaib Arshad

The React.js is preferred by most of the developers when it comes to build front-end and web applications. React.js provide some of the unique features which make it perfect for all kind of applications. Some of the biggest companies in the world use it, including Netflix, Airbnb, Instagram, and Facebook. React.js comprehensive documentation and large community is the key benefit. React.js drawback is that it can be difficult to learn and can be complicated. Its modular design makes it challenging to learn for beginners. The majority of React modules are interconnected, therefore creating a sophisticated application requires the usage of additional programs. Additionally, you must be familiar with functional programming. The terms curry, immutable, and higher-order functions must be known in order to use React.js. You may improve your project by using these React features, which are the most popular ones. Higher-order components and cutting-edge design patterns are features of this complicated language. Learning takes a long time. Beginners can expect to learn the language in about 6 months to 1 year while more professional developers can learn it in a month. React itself does not have a huge learning curve, BUT it assumes that you have a deeper understanding of complex JS topics like ES6, higher-order functions, binding this, map, and a whole bunch of other stuff, not to mention the complexity grenade that Redux throws into the mix. React has very good and detailed documentation, due to its larger community and it is continuously improving day by day [4].

Vue can benefit from all that knowledge needed to learn React.js, but it does not require it and the syntax looks a whole lot more like standard HTML/CSS. A developer with the knowledge of HTML,CSS and JavaScript can learn Vue.js in a week as React.js takes more time to learn. It is easier for the developers to learn the syntax and butt and ifs of Vue.js in less time to make applications more secure and robust [6].

Svelte is fairly a new framework as compared to others and it is gaining popularity quite quickly. It is very fast than React.js as it uses a compiler to convert the code into JavaScript code which runs efficiently. Its build size is very small as well if it's compared to other frameworks. The only drawback of using svelt is that its community is very small as it is a new framework. Other than that it is one of the best front-end framework available today to build web applications without over complicating the code and reducing the resources.

In the final benchmark phase, a DOM removal operation was performed, removing 10,000 <div> elements with all the content. Furthermore, although Vue.js is two times faster than React when updating, it showed the worst results in adding and removal tests.
